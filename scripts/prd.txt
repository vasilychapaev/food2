# Приложение для подсчета калорий и БЖУК

## Цель
Сделать сайт на Laravel, который автоматически собирает данные из Google Sheets (https://docs.google.com/spreadsheets/d/1ds9O-FEElpg8m4jseYO7cHBXXQl3WnjtIYqqAXmPnp8/edit?gid=0#gid=0), считает калории, белки, жиры, углеводы (БЖУК) по каждому приему пищи и за день, и сохраняет результаты в MongoDB.

## Входные данные (Google Sheets)
- Ingredients: список продуктов с алиасами и БЖУК на 100г
- Recipes: рецепты, где ингредиенты смешиваются (например, салаты)
- FoodLog: лог приёмов пищи пользователя (каждая строка — отдельный приём, продукты через точку и вес через точку, например: "грудка.120 - гречка.50")

## Бизнес-логика
- Парсить Google Sheets по cron и по webhook
- Определять новые/изменённые продукты, рецепты, приёмы пищи
- Для каждого приёма пищи:
  - Разбирать строку на продукты и веса
  - Искать продукты по алиасу в Ingredients
  - Если встречается рецепт — раскрывать его ингредиенты и считать итоговый БЖУК
  - Считать итоговые калории, белки, жиры, углеводы по каждому приёму пищи и за день
- Логировать этапы: получение новых данных, расчёты, сохранение

## Хранение
- Все результаты сохранять в MongoDB
- Структура хранения: дни, приёмы пищи, продукты, рассчитанные значения

## API/Контроллеры
- index: список дней с приёмами пищи
- show: подробный отчёт за день — список приёмов пищи, исходная строка, разобранные ингредиенты, рассчитанные БЖУК по каждому приёму и суммарно за день

## Требования
- Laravel
- MongoDB
- Cron/webhook для обновления данных
- Логирование этапов работы 
- docker compose для laravel, mongo. laravel запускается через artisan serve
